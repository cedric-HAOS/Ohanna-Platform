# Ohanna — Brand Identity

Version : 1.0
Statut : Référence officielle

---

# 1. Vision

Ohanna est une plateforme d'observabilité et d'administration d'infrastructures.

Sa mission est de fournir une vision claire, fiable et en temps réel de l'état des services, des équipements et des capacités d'une infrastructure, tout en restant simple à déployer, agréable à utiliser et extensible.

L'identité visuelle doit refléter cette philosophie :

* simplicité ;
* fiabilité ;
* précision ;
* modernité ;
* sobriété.

Chaque élément graphique doit transmettre un sentiment de maîtrise, de stabilité et de confiance.

---

# 2. Valeurs de la marque

L'identité d'Ohanna repose sur cinq valeurs fondamentales.

## Clarté

L'information importante doit être immédiatement identifiable.

L'utilisateur ne doit jamais chercher une information essentielle.

La lisibilité prime toujours sur l'effet visuel.

---

## Fiabilité

L'interface inspire confiance.

Les éléments graphiques sont stables, cohérents et prévisibles.

Aucune animation ou décoration ne doit perturber la compréhension.

---

## Modernité

Ohanna adopte un design contemporain inspiré des outils d'ingénierie modernes.

Le style reste minimaliste, sans surcharge décorative.

---

## Efficacité

Chaque composant possède une fonction claire.

Aucun élément graphique n'est présent uniquement pour décorer.

---

## Cohérence

Tous les produits Ohanna utilisent les mêmes règles graphiques.

L'utilisateur doit retrouver les mêmes repères visuels dans l'ensemble de l'écosystème.

---

# 3. Personnalité visuelle

Ohanna est :

* professionnel ;
* technique ;
* élégant ;
* calme ;
* précis ;
* fiable.

Ohanna n'est pas :

* ludique ;
* agressif ;
* futuriste excessif ;
* "gaming" ;
* chargé visuellement.

---

# 4. Inspirations

Le design s'inspire principalement de :

* interfaces de supervision modernes ;
* tableaux de bord industriels ;
* consoles cloud ;
* outils DevOps contemporains.

L'objectif est d'obtenir une interface intemporelle plutôt qu'un effet de mode.

---

# 5. Signature graphique

L'identité d'Ohanna repose sur trois éléments majeurs.

## Cercle

Le cercle représente :

* la continuité ;
* la surveillance permanente ;
* la résilience.

---

## Réseau

Les connexions évoquent :

* les infrastructures ;
* les services ;
* les dépendances ;
* les échanges.

---

## Nœuds

Les nœuds symbolisent :

* les équipements ;
* les capacités ;
* les services ;
* les points d'observation.

---

# 6. Logo

Le logo officiel est composé :

* du symbole Ohanna ;
* du mot-symbole "Ohanna".

Les deux éléments ne doivent jamais être dissociés sauf pour les icônes ou les favicons.

---

## Variantes

Le dépôt contient les variantes officielles suivantes :

* `logo.svg`
* `logo-dark.svg`
* `logo-light.svg`
* `favicon.svg`
* `favicon.ico`
* `apple-touch-icon.png`
* `icon-192.png`
* `icon-512.png`

---

## Zone de protection

Autour du logo doit être conservé un espace libre équivalent à au moins 25 % de la hauteur du symbole.

Aucun texte ou élément graphique ne peut empiéter sur cette zone.

---

## Taille minimale

Le logo ne doit jamais être affiché à une taille où le symbole perd sa lisibilité.

En dessous de cette taille, utiliser le favicon.

---

# 7. Palette visuelle

L'univers graphique d'Ohanna repose sur des couleurs froides.

Le contraste met en avant les informations importantes sans fatiguer la lecture.

La couleur d'accent officielle est un turquoise lumineux symbolisant l'observabilité.

Les couleurs détaillées sont définies dans **Colors.md**.

---

# 8. Typographie

Ohanna utilise une typographie sans serif moderne.

Les objectifs sont :

* excellente lisibilité ;
* homogénéité ;
* rendu professionnel.

Les règles détaillées sont décrites dans **Typography.md**.

---

# 9. Iconographie

Les icônes doivent être :

* simples ;
* géométriques ;
* cohérentes ;
* facilement reconnaissables.

Une seule bibliothèque d'icônes est utilisée dans l'ensemble de la plateforme.

---

# 10. Illustrations

Les illustrations privilégient :

* les réseaux ;
* les connexions ;
* les infrastructures ;
* les flux de données ;
* les graphes ;
* les schémas techniques.

Les illustrations figuratives ou décoratives sont à éviter.

---

# 11. Animations

Les animations servent uniquement à améliorer la compréhension de l'interface.

Elles doivent être :

* rapides ;
* discrètes ;
* fluides.

Les animations spectaculaires ou distrayantes sont proscrites.

---

# 12. Ton de l'interface

L'interface communique de manière :

* factuelle ;
* concise ;
* professionnelle.

Les formulations ambiguës ou humoristiques sont évitées.

Les messages d'erreur expliquent clairement le problème et proposent une action lorsque cela est possible.

---

# 13. Accessibilité

Le design d'Ohanna vise à être accessible.

Les principes suivants doivent être respectés :

* contraste élevé ;
* lisibilité optimale ;
* navigation clavier ;
* indicateurs de focus visibles ;
* états visuels clairement différenciés.

La couleur seule ne doit jamais porter une information critique.

---

# 14. Cohérence entre produits

Les produits suivants partagent la même identité visuelle :

* Ohanna-Agent
* Ohanna-Vision
* futurs composants de la plateforme

Le Design System constitue la source unique de vérité pour tous les développements.

---

# 15. Usages interdits

Le logo ne doit jamais être :

* étiré ;
* déformé ;
* pivoté ;
* recoloré ;
* ombré ;
* entouré d'effets lumineux ;
* intégré dans une autre forme graphique.

Il est interdit de modifier les proportions du symbole ou du mot-symbole.

---

# 16. Principe fondamental

Chaque décision graphique prise au sein du projet doit répondre à une question simple :

> Cette interface aide-t-elle réellement l'utilisateur à comprendre l'état de son infrastructure plus rapidement ?

Si la réponse est non, cette décision doit être remise en question.

La simplicité, la cohérence et la lisibilité priment toujours sur l'effet visuel.
